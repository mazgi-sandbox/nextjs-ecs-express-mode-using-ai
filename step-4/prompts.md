# Prompts to grow step-4 into step-5

- Add Prisma to the backend. Create `prisma/schema.prisma` with PostgreSQL datasource and Prisma client generator. Create `prisma/item.prisma` with an Item model (id, name, createdAt, updatedAt). Add a `prisma.config.ts` for multi-file schema support. Create `src/prisma/prisma.service.ts` and `src/prisma/prisma.module.ts` as a shared Prisma client module. Add `@prisma/client`, `@prisma/adapter-pg`, `pg`, `dotenv` to backend dependencies and `prisma` to devDependencies. Add prisma scripts (generate, migrate, push, studio) to package.json.
- Create an Items CRUD feature in the backend under `src/items/`. Include: `items.module.ts`, `items.controller.ts` (POST /, GET /, DELETE /:id — public endpoints with Swagger decorators), `items.service.ts` (using PrismaService), `dto/create-item.dto.ts` (name field with class-validator), and `entities/item.entity.ts`. Import ItemsModule in AppModule. Add `class-transformer`, `class-validator`, `@nestjs/swagger` to dependencies.
- Create an Items page in the web frontend at `web/app/app/items/page.tsx`. It should be a `'use client'` component that fetches items from `/backend/items`, displays them in a list, allows adding new items via a form (POST), and deleting items (DELETE). Use direct `fetch()` calls — no auth required. Add a link from the home page to `/items`.
- Add a `db` service (PostgreSQL 17 Alpine) to compose.yaml with a healthcheck. Make the backend service depend on `db` being healthy. Add Prisma commands (`prisma generate && prisma db push --accept-data-loss`) to the backend startup script before `pnpm dev`.
- Add a `mailpit` service to compose.yaml for local email testing (ports 8025 for UI, 1025 for SMTP). Make backend depend on mailpit.
- Create `.example.secrets.env` with `DATABASE_URL` (PostgreSQL connection string for local db), `AUTH_JWT_SECRET`, `AUTH_JWT_REFRESH_SECRET`, `AUTH_SESSION_SECRET`, and `TF_VAR_database_password`. Update compose.yaml to load `.secrets.env` via `env_file` for backend and web services.
- Add Terraform resources for RDS PostgreSQL: `iac/aws/rds.tf` (db_subnet_group + db_instance in private subnets), `iac/aws/secret-versions.tf` (auto-populate DATABASE_URL from RDS endpoint), `iac/aws/secrets-manager.tf` (5 secrets: DATABASE_URL, JWT_SECRET, JWT_REFRESH_SECRET, SESSION_SECRET, SMTP_PASS with IAM policy for ECS). Add RDS security group (port 5432) to vpc-network.tf. Add database variables (password, instance class, name, user) and SMTP variables to variables.tf. Update ecs-express-backend.tf to pass JWT, SMTP environment variables and all secrets.
- Add a backend items e2e test at `backend/test/items.e2e-spec.ts` that tests the items CRUD endpoints. Add `test/jest-e2e.json` and `test/setup.ts` for test configuration. Keep the items e2e test in `web/e2e-tests/tests/items.spec.ts`.
- Update the IaC workflow (`_reusable-iac.yaml`) to pass database_password, SMTP, and other new TF_VARs. Update e2e test workflows to include DATABASE_URL and auth secrets in .env/.secrets.env setup.
